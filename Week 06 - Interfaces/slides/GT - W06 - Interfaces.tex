	\begin{frame}{Simultaneous Inheritance in C\#}
		\begin{itemize}
			\item The catch is: Simultaneous inheritance in C\# is not a thing.
			\begin{itemize}
				\item This is also a trend in other OO languages like Java.
			\end{itemize}
			\item So, how are we going to implement such cases?
			\item One approach mocking multiple inheritance is keep instances of parent classes as fields inside the child class.
			\item But, as you might suspect, this goes against any OOP principle, in practice\footnote{However, this is a way to implement OO in languages that do not support it, like C.\\[1em]}.
		\end{itemize}
	\end{frame}
	
	\begin{frame}{C\# Interfaces}
		\begin{itemize}
			\item One way to implement multiple inheritance in C\# is by using \textbf{Interfaces}.
			\item Interfaces roughly serve the purpose of blueprints, much like abstract classes, describing what a class implementing that interface should comprise of.
			\item As blueprints, Interfaces do not implement themselves any sort of functionality.
			\item Instead, this is left to each class implementing the interface separately to do so.
		\end{itemize}
	\end{frame}
